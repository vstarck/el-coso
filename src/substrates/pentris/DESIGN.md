# Pentris — design record

Generated by `new-substrate`. The six design questions this substrate was
scaffolded against (see `docs/guide.md`):

| Q | Question | Answer |
|---|----------|--------|
| Q1 | Render target | **DOM** |
| Q2 | Viewport relation | **Bounded (lens sizes its own host)** |
| Q3 | Storage shape | **Dense / hot-path (typed-array channels)** |
| Q4 | Agency (how the player acts) | **Continuous / held heading or vector** |
| Q5 | Substrate pace | **Autonomous — rAF drives ticks** |
| Q6 | Commit shape | **Per-event (meaningful state change)** |

## What the scaffold gave you

A no-op-but-green package: it compiles, registers, mounts, and ticks a
no-op. The kit imports, `Inputs` shape, storage alloc, commit predicate,
and feature flags are pre-toggled by the answers above.

## What's left (the thinking, not the busywork)

- **engine/** — the real tick (physics), state fields, channels, commit payload.
- **lens/** — the real render + input handling.
- **puzzles/** — replace the placeholder level.
- Trim the template boilerplate carried in comments.
